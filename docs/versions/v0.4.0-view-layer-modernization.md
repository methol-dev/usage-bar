## 主题

v0.3.0 Provider 自主管理 merge 后 SwiftUI 现代化 audit 的第二步：view 层局部重构。审计抓到但不在 v0.3.1 hygiene 范围内的中等风险项集中到本版本：

- `SettingsView.swift` 5 处 `Binding(get:set:)` 改 `$bindable + onChange`（让 service 暴露可写 `@Published`，副作用搬到 view 的 `onChange`）
- `PopoverView.swift` 6 个 `@ViewBuilder private var ...: some View` helper 抽成独立 `View` struct（views.md / performance.md 双重强调）
- `UsageHeatmapView.swift:122-126` `.onAppear { DispatchQueue.main.async { withAnimation(.none) { ... } } }` 三层嵌套改 `.task { ... }`
- `SettingsView.swift:139-145` `DispatchQueue.main.async` 改 `Task { @MainActor in ... }`（swift.md 禁用 GCD）
- `UsageChartView.swift` `chartOverlay { GeometryReader }` 双层套娃彻底切现代 API（去掉外层 GeometryReader、直接用 proxy）

view 层动刀，UI 回归测试要做。立 spec 时先 brainstorm 决定是切一个大 spec 还是再拆。
