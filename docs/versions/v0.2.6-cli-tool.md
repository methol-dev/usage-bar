  v0.2.6 — cli-tool：**永久不实施**（scope-cut）

  作者自治决策（2026-05-11）：不引入 `claude-usage-bar` CLI 工具。

  评估理由：
  - **违反 ADR 0001 / AGENTS.md "Sparkle 是唯一 runtime dep" 约束**：CLI 子命令解析需 swift-argument-parser（Apple 官方包，但仍是新依赖）；自写 argv 解析繁琐且易错
  - **价值边际**：用户已通过 popover 看到 usage + cost；scriptable 访问的真实场景小众（cron / 自动化脚本）
  - **新 SwiftPM target + 文件 + 测试**：medium 工作量，与边际收益不匹配
  - **install symlink 流程繁琐**：Settings UI "Install CLI" 按钮 → 写 `/usr/local/bin` / `/opt/homebrew/bin` 涉及 SMJobBless / sudo 提示，UX 复杂
  - **替代方案充分**：用户可直接用 ccusage（独立 npm 包，已成熟）做本地 JSONL cost 扫描；本 app 仅做 menu bar UI

  与 v0.2.3 cookie-fallback / v0.2.4 cli-pty-fallback 同款保守判断：通过保守路径优先，避免引入边际收益低 + 维护成本高的功能。

  现有 v0.1.x 已实现的功能集（Phase 2 数据厚度目标达成）覆盖 99% 用户场景：
  - menu bar 5h / 7d / extra usage 实时显示 + 趋势 + pace
  - popover hero card / cost card / chart
  - Claude CLI 凭证零配置登录 + 本地 cost + 多账号
  - Sparkle 自动更新 + 双通道
