# v0.7.1 — 全项目排查加固

## 主题

全项目排查与优化（用户直接发起，非 issue 驱动）：四路并行审查（核心逻辑 / UI 与模型 / 脚本 CI / 文档）后，落地经逐条核实的正确性修复、渲染性能优化、CI/脚本加固与文档一致性治理。发版前追加并入渠道面板优雅降级修复（[PR #41](https://github.com/methol-dev/usage-bar/pull/41)）。

## 含 spec

本版本无独立 spec，属于横切的排查加固（[PR #40](https://github.com/methol-dev/usage-bar/pull/40)）+ 追加修复（[PR #41](https://github.com/methol-dev/usage-bar/pull/41)，设计方案经用户会话内确认）。

## 主要变更

- **Swift 正确性**：Claude fetch 重入 guard + `accountSwitchEpoch`；降采样窗口过滤；悬停范围外返回 nil；月度文件损坏先备份；扫描游标同尺寸改写判全读
- **渠道面板优雅降级**（#41）：删除 Claude 未认证的整屏 `NotAuthenticatedView` 接管；Codex unconfigured 且有本地数据源时局部降级（骨架 hero 卡 + 提示卡 + 本地图表照常）；分支决策抽为 `ProviderPanelLayout.showsFullLayout` 纯函数并配套单测
- **Swift 性能**：费用汇总缓存化；`UsageDataPoint` 身份稳定；热力图阈值/模型构建去重；formatter 与定价表排序缓存
- **CI/脚本**：release workflow 首步预检 `SPARKLE_PRIVATE_KEY` + 补 `swift test` + SwiftPM 缓存；build.yml 去冗余构建；build.sh trap 还原快照；mock-server 4 项修复；issue 脚本 `--limit` 与 merge commit 修正
- **文档**：versions/specs 索引补缺与状态同步；28 处链接层级修正；8 个 spec frontmatter YAML 修复；CLAUDE.md/CONTRIBUTING.md 与 v0.5.1 凭证模型对齐
