# v0.2.8 — Codex History & Trend

## 主题

让 Codex tab 往「和 Claude tab 界面/功能一致」再走一步：给 Codex 也加**历史采样持久化** + **趋势箭头** + **额度折线图**。

- 把 v0.2.5/v0.2.3 里目前是 Claude 专属的 `UsageHistoryService`（`history.json` ring buffer）泛化成 per-provider（或新增 Codex 的等价物），让 Codex 的每次 `wham/usage` 拉取也落一个采样点。
- Codex tab 的 Session/Weekly 卡显示趋势箭头（▲▼ + 增量百分点），和 Claude 一样。
- Codex tab 加上额度折线图区块（复用 / 泛化 `UsageChartSectionView`）。
- 因为要持久化历史，Codex 这条路从「只在 popover 打开时惰性拉一次」升级到也参与一个轻量后台采样节奏（不一定和 Claude 的 polling 完全同频，spec 阶段定）。`supportsBackgroundPolling` 相应可能翻成 true（连带 Settings「Primary Provider」可选 Codex）—— spec 阶段权衡。

**不含**：Codex 本地 session JSONL 成本扫描 / 消费热力图（→ v0.2.9）；阈值通知；多账号。

## 含 spec

- `2026-05-12-codex-history-trend` — Codex 历史采样 + 趋势箭头 + 折线图（泛化 UsageHistoryService / UsageChartSectionView）
